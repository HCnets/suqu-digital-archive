# 苏区镇数字化档案 - 版本更新与技术说明

> ⚠️ **规范警告**: 本文档中的历史版本说明禁止删除，仅允许向下追加，以形成完整的技术演进时间树。

## [v0.3.0-beta] - 2026-05-16
### 1. 3D 纵深感强化
- 开启 `MapLibre GL` 的 `3D Terrain` 功能，引入开源 DEM 数据，让苏区镇的自然山体呈现真实的起伏感。
- 开启建筑拉伸 (`fill-extrusion`)，尝试将基础 2D 多边形拔高为 3D 建筑体。

### 2. 多媒体深度档案下钻
- 重构数据结构，加入了历史年份、长文本详情以及图片相册数组。
- 新增 `ArchiveDetailModal`：当用户在侧边栏点击“深度解析”时，弹出一个覆盖全局的 Liquid Glass 模态框，左侧轮播历史老照片，右侧展示详实的历史文献。

### 3. 空间漫游（自动巡航）
- 在顶部 Header 加入了“自动巡航”入口。
- 通过状态管理控制巡航回路，每隔 8 秒镜头自动随机旋转角度并飞向 (`flyTo`) 下一个档案坐标点，极大地增强了大屏展示的“演示感”与“叙事性”。

## [v0.2.0-beta] - 2026-05-16
### 1. 架构升维：真实 GIS 底座
- 在没有真实 3D 资产的情况下，通过 `MapLibre GL JS` 引入了真实的经纬度坐标系底图，代替纯虚构的 3D 空间。
- 将视角初始中心锁定在：广东省紫金县苏区镇 (`longitude: 115.3415`, `latitude: 23.3610`)。

### 2. 交互与 UI 进化
- 移除 Three.js 占位方块，使用基于 DOM 映射地理坐标的 `PoiMarker` 组件。
- 利用 CSS 渐变、`backdrop-blur` 和 `transform` 实现了科技感十足的**发光点位与冲天光柱**。
- 档案玻璃面板（Glass Panel）新增档案类别颜色标识（红/蓝/黄）与真实的经纬度数据显示，增强数字化档案的真实感。

## [v0.1.1-alpha] - 2026-05-16
### 1. 前端工程体系建立
- 采用 React + Vite + TypeScript 搭建极速前端环境。
- 配置 `@/` 路径别名，优化引入结构。

### 2. UI 框架与核心组件
- 成功集成 TailwindCSS 并封装了 `@layer utilities` 的 `.liquid-glass` 与 `.glass-panel` 类。
- 落地 `UnifiedHeader` 组件：严格执行大字号标题 + 右侧分离式玻璃返回按钮设计。
- 落地 3D 地图互动层：使用 `react-three-fiber` 呈现基础的白模和发光 POI 点位。

### 3. 数据流设计
- 引入 Zustand 建立 `useAppStore`。
- 实现地图空间坐标点击事件与前端 UI 数据状态的打通（点击模型 -> 触发状态更新 -> 弹出包含档案信息的玻璃面板）。

## [v0.1.0-alpha] - 2026-05-16
### 1. 架构选型 (GPT-5.4 推荐)
- **WebGIS 引擎**: 推荐使用 Cesium.js 处理宏观地形（DEM）与倾斜摄影模型，或 Three.js 结合 MapboxGL 处理轻量级建筑级交互。
- **前端框架**: React 18 + TypeScript + Vite，保障极速编译与模块化。
- **状态管理**: Zustand（轻量且适合处理复杂的 3D 状态交互）。

### 2. UI 规范 (Gemini 3.1 Pro Max 设定)
- **大一统 Header**: 采用横向流式布局，左侧为大字号标题（headlineMedium），右侧为独立“返回”按钮。
- **返回按钮设计**: 尺寸 20dp icon，12dp padding，带有 Liquid Glass (液态玻璃) 质感。
- **信息面板**: 点击地图上的 3D 建筑/热点时，弹出的档案信息面板必须采用磨砂玻璃效果（backdrop-filter: blur），突出“数字化”与“未来感”。

### 3. 下一步研发计划
- 搭建基础前端工程 (Vite + React)。
- 引入地图引擎底图并定位至“广东省苏区镇”。
- 实现第一个“点击 POI 弹出 Liquid Glass 档案面板”的 Demo。
