# 苏区镇数字化档案 - 版本更新与技术说明

> ⚠️ **规范警告**: 本文档中的历史版本说明禁止删除，仅允许向下追加，以形成完整的技术演进时间树。

## [v1.0.0-rc] - 2026-05-16
### 1. HUD 宏观数据雷达
- 新增 `HudDashboard.tsx` 组件，常驻于屏幕左侧。
- 采用 Glassmorphism 设计，包含“苏区镇档案总览”（动态数字统计、分类进度条）以及“Active Radar”（实时转动的 CSS 动画雷达与虚拟坐标流），弥补了大屏两侧的空旷感。

### 2. 程序化 3D 建筑白模
- 在 `GisMap` 中新增 `poi-3d-buildings` 数据源和 `fill-extrusion` 渲染层。
- 算法根据档案的经纬度坐标，自动向外偏移 `0.0003` 度生成一个多边形底座，并拉伸出 100~150 米高的 3D 实体，颜色与档案类型（红/蓝/黄）强绑定，彻底取代了扁平的 Marker。

### 3. Orbit Fly 环绕盘旋镜头
- 重构了 `isAutoTouring` 逻辑。
- 当自动巡航到达某个档案坐标后，启动 `requestAnimationFrame` 不断微调 `bearing`（方位角），实现镜头围绕该建筑体进行 360 度平滑盘旋的电影级运镜。盘旋 9 秒后自动切入下一个地点。

## [v0.6.0-beta] - 2026-05-16
### 1. 业务形态收敛
- 响应用户“对外展示为主”的需求，移除了前台 Admin 录入中心入口，防止演示时误触。将系统的纯净度拉满。

### 2. 电影级开场动画 (Fly-in)
- 重构了 `GisMap` 的初始化视角，默认将高度拉到外太空 (`zoom: 4`)，并且禁止用户交互。
- 在底图加载完毕后，利用 `flyTo` 方法配合 4000ms 的缓动曲线 (`curve: 1.5`)，实现上帝视角直冲苏区镇的震撼坠入效果，落地后自动恢复用户交互权限。

### 3. 全息沙盘氛围
- 增加了一个 `z-index` 垫在地图之上的全屏粒子遮罩层，使用 CSS `radial-gradient` 和 `mix-blend-screen` 混合模式，给原本偏平的暗黑地图罩上了一层立体的全息投影蓝光。

## [v0.5.0-beta] - 2026-05-16
### 1. 数据录入中心 (Admin Drawer)
- 在右上角 Header 区域新增了 `录入中心` 入口，点击后从右侧滑出带有 Liquid Glass 毛玻璃质感的表单面板。
- 支持录入档案名称、坐标、发生年份、短述以及深度档案内容（预留给富文本）。

### 2. 交互式坐标拾取
- 结合 MapLibre 的 `click` 事件，当侧边栏处于开启状态时，点击地图上的任何一点，都不会触发弹窗，而是会直接将精确到小数点后 6 位的 `lng/lat` 数据回填到表单中。

### 3. Zustand Persist 本地持久化
- 修改了 `store/index.ts`，利用 `zustand/middleware` 中的 `persist` 模块。
- 采用 `partialize` 技术，仅将 `archives` 对象树持久化保存至 `localStorage`，而将 UI 状态（如弹窗是否打开、当前时间轴等）排除在外。保证了页面刷新后录入的数据不会丢失。

## [v0.4.0-beta] - 2026-05-16
### 1. 全局时间沙盘 (Time Slider)
- 在页面底部新增了横向的时间沙盘组件。
- 将档案数据与时间轴进行强绑定。用户拖拽年份时，地图上只会显示在该年份之前（含该年份）已经存在的历史档案，呈现出随历史时间演进的“星火燎原”效果。

### 2. 多源底图切换 (Map Switcher)
- 右侧新增悬浮的 Liquid Glass 图层切换器。
- 提供“科技暗黑风”与“高清卫星实景”两种底图引擎的无缝切换。通过 React 的 Key Re-mount 机制，完美解决了 MapLibre 底图切换时的上下文污染问题。

### 3. GeoJSON 边界高亮
- 绘制了苏区镇的核心行政边界，使用半透明填充 (`fill-opacity: 0.1`) 和边缘发光线条 (`line-color`) 强化了“数字版图”的概念。

## [v0.3.0-beta] - 2026-05-16
### 1. 3D 纵深感强化
- 开启 `MapLibre GL` 的 `3D Terrain` 功能，引入开源 DEM 数据，让苏区镇的自然山体呈现真实的起伏感。
- 开启建筑拉伸 (`fill-extrusion`)，尝试将基础 2D 多边形拔高为 3D 建筑体。

### 2. 多媒体深度档案下钻
- 重构数据结构，加入了历史年份、长文本详情以及图片相册数组。
- 新增 `ArchiveDetailModal`：当用户在侧边栏点击“深度解析”时，弹出一个覆盖全局的 Liquid Glass 模态框，左侧轮播历史老照片，右侧展示详实的历史文献。

### 3. 空间漫游（自动巡航）
- 在顶部 Header 加入了“自动巡航”入口。
- 通过状态管理控制巡航回路，每隔 8 秒镜头自动随机旋转角度并飞向 (`flyTo`) 下一个档案坐标点，极大地增强了大屏展示的“演示感”与“叙事性”。

## [v0.2.0-beta] - 2026-05-16
### 1. 架构升维：真实 GIS 底座
- 在没有真实 3D 资产的情况下，通过 `MapLibre GL JS` 引入了真实的经纬度坐标系底图，代替纯虚构的 3D 空间。
- 将视角初始中心锁定在：广东省紫金县苏区镇 (`longitude: 115.3415`, `latitude: 23.3610`)。

### 2. 交互与 UI 进化
- 移除 Three.js 占位方块，使用基于 DOM 映射地理坐标的 `PoiMarker` 组件。
- 利用 CSS 渐变、`backdrop-blur` 和 `transform` 实现了科技感十足的**发光点位与冲天光柱**。
- 档案玻璃面板（Glass Panel）新增档案类别颜色标识（红/蓝/黄）与真实的经纬度数据显示，增强数字化档案的真实感。

- 实现地图空间坐标点击事件与前端 UI 数据状态的打通（点击模型 -> 触发状态更新 -> 弹出包含档案信息的玻璃面板）。

## [v0.1.0-alpha] - 2026-05-16
### 1. 架构选型 (GPT-5.4 推荐)
- **WebGIS 引擎**: 推荐使用 Cesium.js 处理宏观地形（DEM）与倾斜摄影模型，或 Three.js 结合 MapboxGL 处理轻量级建筑级交互。
- **前端框架**: React 18 + TypeScript + Vite，保障极速编译与模块化。
- **状态管理**: Zustand（轻量且适合处理复杂的 3D 状态交互）。

### 2. UI 规范 (Gemini 3.1 Pro Max 设定)
- **大一统 Header**: 采用横向流式布局，左侧为大字号标题（headlineMedium），右侧为独立“返回”按钮。
- **返回按钮设计**: 尺寸 20dp icon，12dp padding，带有 Liquid Glass (液态玻璃) 质感。
- **信息面板**: 点击地图上的 3D 建筑/热点时，弹出的档案信息面板必须采用磨砂玻璃效果（backdrop-filter: blur），突出“数字化”与“未来感”。
